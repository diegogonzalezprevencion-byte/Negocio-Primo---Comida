LEER ANTES DE SUBIR A GITHUB

Esta versión incorpora un backroom de administración para Chef Oscar.

IMPORTANTE:

- No incluye package.json.
- No incluye package-lock.json.
- No incluye node_modules.
- No usa Next.js ni React.
- No requiere npm install.
- No requiere build.
- No incluye 404.html.

Archivo principal del sitio:

index.html

Backroom de administración:

backroom.html
PIN inicial: 1234

Desde el backroom puedes modificar platos, precios, vigencia diaria/semanal/mensual y marcar si un plato está disponible o no disponible.

NOTA TÉCNICA:
Al ser una web estática, los cambios se guardan en el navegador mediante localStorage. Se actualizan en tiempo real en pestañas abiertas del mismo navegador. Para que el cambio sea visible automáticamente para todos los visitantes en internet, se requiere una base de datos o backend.

Pasos recomendados:

1. Borra todo el contenido anterior del repositorio.
2. Sube solamente los archivos internos de este ZIP.
3. No subas la carpeta completa como carpeta; sube el contenido interno.
4. Asegúrate de que index.html esté directamente en la raíz.
5. En Vercel, usa Framework Preset: Other.
6. Deja Install Command y Build Command vacíos.
7. Output Directory: ./
