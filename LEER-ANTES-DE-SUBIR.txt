LEER ANTES DE SUBIR A GITHUB

Esta versión ajusta el sitio estático anterior a un negocio de comidas caseras a domicilio: Chef Oscar.

IMPORTANTE:

- No incluye package.json.
- No incluye package-lock.json.
- No incluye node_modules.
- No usa Next.js ni React.
- No requiere npm install.
- No requiere build.
- No incluye 404.html para evitar que GitHub muestre primero una página 404.

El archivo principal del sitio es:

index.html

Debe quedar en la raíz del repositorio.

Pasos recomendados:

1. Borra todo el contenido anterior del repositorio.
2. Sube solamente los archivos internos de este ZIP.
3. No subas la carpeta completa como carpeta; sube el contenido interno.
4. Asegúrate de que index.html esté directamente en la raíz.
5. En Vercel, usa Framework Preset: Other.
6. Deja Install Command y Build Command vacíos.
7. Output Directory: ./

Se dejaron páginas de redirección para destinos.html, cotizar.html y experiencias.html, de modo que enlaces antiguos lleven al nuevo contenido de menú, pedidos y promociones.
